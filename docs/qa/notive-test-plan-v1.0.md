# 테스트 계획서 v1.0

# Notive Test Plan

---

# 1. 문서 목적

본 문서는 Notive MVP 출시 전 품질 검증을 위한 테스트 계획을 정의한다.

이 문서는 단위 테스트 코드나 자동화 스크립트가 아니다. 기능 테스트, 권한 테스트, AI 품질 테스트, 검색 테스트, 관리자/운영 테스트, 출시 차단 기준을 정리하여 QA와 개발 검증의 기준으로 사용한다.

---

# 2. 테스트 목표

MVP 테스트의 목표는 다음과 같다.

* 핵심 사용자 흐름이 중단 없이 동작하는지 확인한다.
* 문서, 검색, AI 생성에서 권한 없는 데이터가 노출되지 않는지 확인한다.
* 문서 저장과 버전 관리가 안정적으로 동작하는지 확인한다.
* AI 생성 결과가 업무 문서 초안으로 사용할 수 있는지 확인한다.
* 검색 결과와 AI 요약이 출처 기반으로 동작하는지 확인한다.
* 관리자 기능이 조직 운영에 필요한 최소 수준을 만족하는지 확인한다.
* 출시를 막아야 할 이슈와 출시 후 개선 가능한 이슈를 구분한다.

# 16. 출시 차단 기준

다음 문제가 있으면 MVP 출시를 보류한다.

* 다른 조직 데이터 접근 가능
* 권한 없는 문서 조회 가능
* 권한 없는 문서가 검색 결과에 노출
* 권한 없는 문서가 AI 참고 자료로 사용됨
* 문서 저장 시 데이터 유실
* 로그인 불가
