# 08. D단계 AI 문서 생성 계획서 작업 보고서

## 대상 문서

`docs/implementation/notive-implementation-plan-d-ai-document-generation-v1.0.md`

D단계(AI 문서 생성)에서 무엇을 어떻게 만들지 정의한 세부 계획서다. AI에게 문서 초안을 요청하고, 결과를 받아서 사용자가 편집/저장하는 흐름이 여기서 다뤄진다.

## 변경한 내용

### 1. §7.2 AIRequest 데이터 모델을 메타데이터 전용으로 축소

기존 주요 필드: ID, 조직, 요청 사용자, 문서 유형, 템플릿, **요청 내용**, 문서 목적, 대상 독자, 톤앤매너, 상태, 생성/완료일.

변경 후: ID, 조직, 요청 사용자, 문서 유형, 템플릿, 문서 목적(태그), 대상 독자(태그), 톤앤매너(태그), 상태, **입출력 토큰 수**, **결과 저장 여부**, 생성/완료일.

핵심: **"요청 내용"(사용자가 AI에게 보낸 문장)을 영구 저장 필드에서 제거**. 대신 토큰 사용량과 결과 저장 여부 같은 분석용 메타데이터를 추가했다.

문서 위에 메모 추가: "MVP에서 AIRequest는 메타데이터만 영구 저장한다. 사용자 요청 원문은 영구 저장하지 않는다(Phase A §15, DB 설계 §9.1)."

비저장 필드 표기: "요청 원문 — opt-in 문제 신고 시 `ai_request_payloads`에 30일 저장."

### 2. §7.3 AIResult 데이터 모델을 메타데이터 전용으로 축소

기존 주요 필드: ID, AI 요청 ID, **제목**, **본문**, 결과 상태, 저장된 문서 ID, **오류 메시지**, 생성일.

변경 후: ID, AI 요청 ID, 결과 상태, 저장된 문서 ID, **오류 코드**, 생성일.

핵심: **제목, 본문, 자유 형식 오류 메시지를 모두 영구 저장 필드에서 제거**. 자유 메시지 대신 코드값만 저장한다.

비저장 필드 표기: 제목/본문은 "세션 바운드 단기 스토리지에서 미저장 24시간 또는 폐기 시 삭제. opt-in 문제 신고 시 `ai_request_payloads`에 30일 저장."

## 변경 이유

- **DB 설계와 정합**: DB 설계서(§9.1, §9.2)에서 이미 같은 칸들을 영구 테이블에서 제거했다. D단계 계획서가 옛 모델을 가리키고 있으면 개발자가 어느 쪽을 따라야 하는지 헷갈린다.
- **Phase A §15 직접 반영**: AI 본문은 회사 내부 정보가 그대로 담긴다. 영구 저장하지 않는다는 결정이 데이터 모델 수준에서 보이지 않으면 구현 시 무심코 저장하게 된다.
- **자유 오류 메시지를 코드로 바꾼 이유**: 자유 메시지는 종종 사용자 입력을 그대로 담는다. 코드값만 저장하면 메시지에 민감 정보가 섞일 위험이 사라진다. 사람이 읽을 메시지는 코드를 다국어 사전에서 조회해 화면에 띄운다.

## B단계 착수에 미치는 영향

- D단계는 B단계 후 두 단계나 떨어져 있지만, **B단계 인프라 결정에 영향**을 준다. AI 결과 본문이 영구 DB에 안 들어가므로 단기 스토리지(Redis 등)와 24시간 만료 처리 인프라가 B단계 인프라 작업에 포함되어야 한다.
- B단계에서 AI 호출 골격(D단계 본격 구현 전 토대)을 만들 때, "본문은 저장 안 한다"는 전제로 코드를 짜야 한다.

## 남은 주의사항

- §7.3에 남아 있는 "결과 상태" 값들(Generated, Selected, Saved, Discarded, Failed)이 카멜케이스인데, 이번 작업에서는 손대지 않았다. To-do 상태처럼 소문자로 통일할지 D단계 진입 시 결정 필요.
- 본문이 영구 저장되지 않으므로 AI 생성 품질 회귀 분석이 어려워진다. opt-in 신고 흐름이 충분한 분석 데이터를 제공하는지 운영 시작 후 점검이 필요하다.
- 문서에 `purpose`/`audience`/`tone`을 "구조 태그"로 분류했는데, 사용자가 자유 텍스트로 입력하는 형태라면 이것도 사실상 사용자 입력 본문이 될 수 있다. D단계 화면 설계 시 **선택지 기반 입력**(드롭다운 등)으로 정형화할지, 자유 텍스트로 둘지 결정이 필요하다.
