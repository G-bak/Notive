# PRD (Product Requirements Document)

# Notive: AI 기반 사내 문서 및 업무 운영 플랫폼

---

# 1. 프로젝트 개요

## 프로젝트명

Notive

- Note + Active
- 업무 기록을 실제 실행 가능한 문서와 지식 자산으로 연결하는 플랫폼

---

## 한 줄 정의

Notive는 기업 내부의 문서, 업무 기록, 회의 내용, 프로젝트 맥락을 기반으로 직원이 더 빠르고 일관되게 업무 문서를 만들고 공유하도록 돕는 AI 업무 운영 플랫폼이다.

---

## 프로젝트 목적

AI를 활용하여 기업 내 문서 제작과 업무 기록 과정을 통합 관리함으로써 다음 문제를 해결한다.

* 반복적인 문서 작성 부담 감소
* 직원별 문서 품질 편차 완화
* 업무 기록과 문서의 중앙화
* 팀 단위 협업과 승인 흐름 개선
* 사내 지식 검색과 재사용 효율 향상

본 제품은 단순히 프롬프트를 입력해 문서를 생성하는 도구가 아니라, 직원의 업무 활동과 조직 내 업무 흐름을 이해하여 실제 업무 맥락에 맞는 문서를 생성하고 관리하는 플랫폼을 지향한다.

---

# 2. 제품 배경

## 기존 업무 환경의 문제

많은 기업에서는 문서 작성, 업무 기록, 회의 정리, 프로젝트 공유가 여러 도구에 분산되어 있다. 이로 인해 직원은 이미 존재하는 정보를 반복해서 정리해야 하고, 팀 리더는 구성원별 문서 품질과 보고 방식의 편차를 관리해야 한다.

주요 문제는 다음과 같다.

* 반복적인 보고서, 회의록, 제안서 작성에 많은 시간이 소요됨
* 담당자마다 문서 양식과 표현 방식이 달라 조직 차원의 일관성이 떨어짐
* 프로젝트 히스토리와 의사결정 근거가 여러 곳에 흩어짐
* 기존 사내 문서를 찾기 어렵고 재사용률이 낮음
* 회의 내용과 To-do가 문서, 일정, 업무 관리 도구에 분산됨
* 신규 직원이나 타 부서 구성원이 업무 맥락을 빠르게 파악하기 어려움

---

## 목표하는 업무 환경

Notive는 다음 데이터를 하나의 업무 맥락으로 연결한다.

* 기존 사내 문서
* 업무 기록
* 프로젝트 상태
* 회의 내용
* 일정
* To-do
* 조직 및 팀 단위 협업 데이터

이를 통해 사용자는 “문서를 새로 작성한다”기보다 “이미 존재하는 업무 맥락을 바탕으로 필요한 문서를 완성한다”는 경험을 갖게 된다.

---

# 3. 제품 비전

## 제품 비전

Notive는 회사의 업무 흐름을 이해하고, 그 흐름을 문서와 지식 자산으로 전환하는 AI 업무 운영 플랫폼이 된다.

---

## 제품 차별점

기존 AI 문서 도구는 사용자의 프롬프트와 제한적인 입력값을 기반으로 문서를 생성하는 경우가 많다. Notive는 회사 내부의 업무 기록과 기존 문서 맥락을 함께 활용하여 더 실제적인 결과물을 제공하는 것을 목표로 한다.

| 구분 | 기존 AI 문서 도구 | Notive |
| --- | --- | --- |
| 입력 방식 | 사용자가 매번 설명 입력 | 업무 기록과 사내 문서를 함께 활용 |
| 결과물 | 일반적인 문서 초안 | 회사 맥락이 반영된 업무 문서 |
| 지식 활용 | 개별 요청 중심 | 조직 지식 축적 및 재사용 |
| 협업 | 개인 생산성 중심 | 팀 공유, 승인, 권한 관리 중심 |
| 장기 가치 | 일회성 생성 | 업무 히스토리와 지식 자산화 |

---

# 4. 타깃 사용자

## 1차 타깃

초기 타깃은 문서 작성과 업무 보고 부담이 큰 중소기업(SMB)이다.

특히 다음과 같은 조직에 적합하다.

* 반복적인 보고서, 제안서, 회의록 작성이 많은 조직
* 문서 양식과 품질을 표준화하고 싶은 조직
* 팀별 업무 기록과 프로젝트 히스토리가 분산되어 있는 조직
* 사내 문서 검색과 재사용 효율이 낮은 조직
* Notion, Google Docs, Slack, 메신저 등 여러 도구에 업무 정보가 흩어져 있는 조직

---

## 주요 사용자 유형

| 사용자 유형 | 주요 니즈 | 대표 사용 시나리오 |
| --- | --- | --- |
| 일반 직원 | 문서 작성 시간 단축, 업무 기록 정리 | 주간보고, 회의록, 제안서 초안 생성 |
| 팀 리더 | 팀 문서 품질 관리, 승인 및 공유 | 팀 보고서 검토, 문서 공유 범위 관리 |
| 관리자 | 사용자, 권한, 조직 설정 관리 | 부서별 접근 권한 설정, 감사 로그 확인 |
| 임원진 | 조직 현황과 주요 리포트 확인 | 프로젝트 요약, 주요 이슈 리포트 조회 |

# 5. 핵심 사용 흐름

## 흐름 1. 주간 업무 보고서 생성

1. 직원이 “이번 주 업무보고 작성해줘”라고 요청한다.
2. Notive는 해당 직원의 업무 기록, 완료된 To-do, 최근 회의 내용, 프로젝트 상태를 참고한다.
3. AI가 회사 양식에 맞는 주간 보고서 초안을 생성한다.
4. 직원은 초안을 수정하고 저장한다.
5. 팀 리더는 공유된 보고서를 검토하거나 승인한다.

---

## 흐름 2. 회의록 및 액션 아이템 정리

1. 사용자가 회의 내용을 입력하거나 회의 메모를 업로드한다.
2. Notive는 논의 주제, 결정 사항, 후속 업무를 분리한다.
3. 회의록 문서와 To-do 초안을 생성한다.
4. 담당자와 마감일을 지정하여 팀과 공유한다.

---

## 흐름 3. 기존 자료 기반 제안서 초안 생성

1. 사용자가 “A 고객사 제안서 초안 작성해줘”라고 요청한다.
2. Notive는 유사한 과거 제안서, 관련 프로젝트 자료, 고객 미팅 메모를 참고한다.
3. 회사 템플릿에 맞는 제안서 구조와 주요 내용을 생성한다.
4. 직원은 필요한 내용을 보완한 뒤 문서로 저장하고 공유한다.

# 7. 핵심 기능 요구사항

## 기능 1. AI 문서 생성

직원은 자연어 요청을 통해 업무 문서를 생성할 수 있다.

### 지원 문서 유형

* 주간/월간 업무 보고서
* 회의록
* 제안서
* 기획서
* SOP
* 이메일 초안
* 기술 문서
* 정책 문서

### 주요 요구사항

* 자연어 요청 기반 문서 초안 생성
* 회사 또는 부서별 템플릿 적용
* 문체, 맞춤법, 표현 방식 보정
* 제목, 목차, 표, 목록 등 기본 문서 구조 생성
* 생성 결과를 사용자가 수정 가능한 형태로 제공
* 생성된 문서의 저장 및 공유 연결

---

## 기능 2. 업무 맥락 기반 문서 생성

Notive는 단순 입력값뿐 아니라 업무 맥락을 참고하여 문서를 생성한다.

### 참고 가능한 정보

* 기존 문서
* 업무 다이어리
* 프로젝트 상태
* 회의 내용
* 일정
* To-do

### 주요 요구사항

* 사용자가 명시적으로 선택한 자료를 문서 생성에 반영
* 관련성이 높은 기존 문서를 추천
* 업무 기록을 기반으로 보고서, 회의록, 요약문 생성
* AI가 참고한 주요 출처를 사용자가 확인 가능
* 사용자가 원하지 않는 정보는 생성 맥락에서 제외 가능

---

## 기능 3. 중앙 문서 저장 및 공유

모든 문서는 중앙 저장소에서 관리되며, 사용자는 권한에 따라 문서를 조회, 수정, 공유할 수 있다.

### 주요 요구사항

* 생성 문서 자동 저장
* 임시 저장 및 초안 관리
* 문서 수정 이력 관리
* 이전 버전 확인 및 복원
* 개인, 팀, 부서, 프로젝트 단위 공유
* 문서 즐겨찾기 및 최근 문서 확인
* 제목, 작성자, 태그, 내용 기반 검색

---

## 기능 4. 권한 및 접근 제어

조직 구조와 사용자 역할에 따라 문서 접근 범위를 제어한다.

### 권한 유형

| 권한 | 설명 |
| --- | --- |
| Viewer | 문서 조회 가능 |
| Editor | 문서 작성 및 수정 가능 |
| Manager | 팀 문서 승인 및 공유 관리 가능 |
| Admin | 사용자, 조직, 권한 설정 관리 가능 |

### 주요 요구사항

* 부서별 문서 접근 제한
* 특정 문서의 열람, 수정, 공유 권한 설정
* 외부 공유 제한
* 다운로드 허용 여부 설정
* 주요 활동에 대한 감사 로그 기록
* AI 검색 결과에서도 사용자의 권한 범위를 준수

---

## 기능 5. 사내 지식 검색

사용자는 사내 문서를 자연어로 검색하고 필요한 내용을 빠르게 찾을 수 있다.

### 주요 사용 예시

* “작년 보안 정책 문서 찾아줘”
* “유사한 고객사 제안서 보여줘”
* “지난 분기 마케팅 보고서 요약해줘”
* “A 프로젝트에서 결정된 주요 이슈 정리해줘”

### 주요 요구사항

* 제목, 내용, 태그 기반 검색
* 자연어 기반 문서 검색
* 검색 결과의 원문 문서 연결
* 검색 결과 요약 제공
* 사용자가 접근 권한이 있는 문서만 검색 결과에 노출
* 답변에 활용된 문서 출처 표시

---

## 기능 6. 업무 다이어리

사용자는 자신의 업무 활동을 짧게 기록하고, 해당 기록을 이후 문서 생성에 활용할 수 있다.

### 기록 예시

* 고객사 미팅 진행
* API 오류 수정 완료
* 디자인 수정 요청 접수
* 계약 조건 검토 완료

### 주요 요구사항

* 날짜별 업무 기록 작성
* 프로젝트 또는 태그 연결
* 관련 문서 연결
* 업무 기록 기반 보고서 생성
* 개인 기록과 팀 공유 기록 구분

---

## 기능 7. 일정 및 To-do 관리

업무 일정과 할 일을 문서 생성 맥락으로 활용할 수 있도록 관리한다.

### 주요 요구사항

* 개인 일정과 팀 일정 관리
* 프로젝트 일정과 마감일 관리
* To-do 생성, 담당자 지정, 완료 상태 관리
* 우선순위 설정
* 완료 업무와 미완료 업무를 보고서 생성에 반영

---

## 기능 8. 관리자 기능

관리자는 조직의 사용자, 권한, 템플릿, 사용 현황을 관리한다.

### 주요 요구사항

* 사용자 초대 및 비활성화
* 부서 및 팀 관리
* 역할 및 권한 설정
* 문서 템플릿 관리
* 사용량 및 주요 활동 현황 확인
* 감사 로그 조회

# 8. MVP 범위

## MVP 원칙

MVP는 “회사의 업무 맥락을 반영한 AI 문서 생성이 실제로 유용한가”를 검증하는 데 집중한다.

초기에는 모든 업무 자동화를 구현하기보다, 반복 문서 작성과 사내 문서 재사용 문제를 먼저 해결한다.

---

## 1차 MVP 포함 범위

* 사용자 로그인
* 기본 역할 및 권한 관리
* AI 문서 생성
* 회사/부서 템플릿 적용
* 문서 저장 및 공유
* 문서 검색
* 업무 다이어리
* 기본 관리자 기능

---

## 1차 MVP에서 제한적으로 제공하는 범위

* 사내 지식 검색은 제한된 문서 유형부터 지원
* 일정 및 To-do는 문서 생성에 필요한 최소 기능 중심으로 제공
* 권한 관리는 조직, 부서, 문서 단위의 기본 제어부터 제공
* AI가 참고한 자료의 출처 표시는 핵심 문서 생성 흐름에서 우선 제공

---

## 2차 개발 범위

* 고도화된 일정 및 To-do 관리
* 실시간 음성 회의록 생성
* 외부 협업 도구 연동
* AI Agent 기반 업무 자동 실행
* 모바일 앱
* Desktop App
* On-Premise 및 VPC 지원
* 엔터프라이즈 보안 및 관리 기능 고도화

---

# 9. 플랫폼 전략

## 초기 플랫폼

초기에는 Web 기반 SaaS로 제공한다.

### 이유

* 별도 설치 없이 빠르게 도입 가능
* SMB 고객의 도입 장벽이 낮음
* 제품 개선과 배포가 빠름
* 고객 피드백을 빠르게 수집 가능

### 지원 환경

* Chrome
* Edge
* Safari
* Firefox

---

## 확장 플랫폼

엔터프라이즈 고객 요구가 확인되면 다음 형태로 확장한다.

* Desktop App
* On-Premise
* VPC
* 기업 전용 보안 환경
* 내부망 또는 폐쇄망 대응

# 10. 보안 및 신뢰 요구사항

Notive는 사내 문서와 업무 기록을 다루므로 보안과 신뢰가 핵심 제품 요구사항이다.

## 주요 요구사항

* 회사별 데이터 분리
* 사용자 권한에 따른 문서 접근 제어
* AI 검색 및 문서 생성 시 권한 범위 준수
* 중요 활동에 대한 감사 로그 기록
* 문서 삭제 및 보존 정책 제공
* 외부 공유 및 다운로드 제한 옵션 제공
* 고객 데이터의 AI 학습 사용 여부를 명확히 고지
* 관리자가 조직 내 데이터 사용 정책을 설정 가능

# 12. 성공 지표

## 제품 사용 지표

* 주간 활성 사용자 수
* 사용자당 문서 생성 횟수
* 생성된 문서의 저장률
* AI 초안 사용 후 수정 완료율
* 문서 공유 횟수
* 검색 사용 횟수

* 반복 문서 작성 시간 감소율

* 기존 문서 재사용률
* 팀 리더의 문서 검토 시간 감소
* 사용자의 AI 생성 결과 만족도

* 검색 결과 불만족 피드백 비율
* 권한 관련 오류 발생 건수
* 문서 저장 및 복원 실패 건수

# 14. 제품 메시지

Notive는 다음 메시지를 중심으로 시장에 전달한다.

> 문서를 새로 쓰는 시간이 아니라, 이미 진행한 업무를 정리하는 시간을 줄인다.

> 흩어진 업무 기록을 회사의 문서와 지식 자산으로 바꾼다.

> 단순한 AI 문서 생성기가 아니라, 회사의 업무 흐름을 이해하는 AI 운영 플랫폼이다.
