# Notive 안정화 및 출시 준비

---

# 1. 문서 목적

본 문서는 Notive 전체 구현 계획 중 H단계인 안정화 및 출시 준비의 세부 계획을 정의한다.

H단계의 목적은 A-G 단계에서 구현한 기능을 초기 고객 또는 내부 테스트 그룹에 제공 가능한 수준으로 안정화하고, 출시 전 품질, 보안, 운영, 피드백 수집 체계를 준비하는 것이다.

---

# 2. H단계 목표

H단계가 완료되면 다음이 가능해야 한다.

* 핵심 사용자 흐름이 중단 없이 동작한다.
* 인증, 권한, 문서 접근, AI 생성, 검색의 주요 리스크가 점검된다.
* 출시 전 테스트 체크리스트가 통과된다.
* 기본 운영 정책과 장애 대응 기준이 준비된다.
* 초기 사용자 피드백을 수집하고 분류할 수 있다.
* 알려진 이슈와 출시 제외 범위가 정리된다.
* 초기 배포와 롤백 기준이 준비된다.

| 흐름 | 검증 내용 |
| --- | --- |
| 신규 조직 시작 | 가입, 조직 생성, Admin 부여 |
| 사용자 초대 | 초대 발송, 수락, 역할 부여 |

| 문서 검색 | 제목/본문/필터 검색 |
| AI 요약 검색 | 출처 기반 요약 표시 |
| 관리자 운영 | 사용자, 팀, 템플릿, 로그 조회 |

---

## 6.2 권한 테스트

### 필수 검증

* Viewer는 문서를 작성할 수 없다.
* Viewer는 AI 문서 생성에 접근할 수 없다.
* Editor는 관리자 화면에 접근할 수 없다.
* Manager는 허용된 팀 범위만 관리할 수 있다.
* Admin은 조직 설정과 사용자 관리를 수행할 수 있다.
* 다른 조직의 문서는 조회할 수 없다.
* 권한 없는 문서는 검색 결과에 표시되지 않는다.
* 권한 없는 문서는 AI 참고 자료로 사용되지 않는다.

---

## 6.3 문서 데이터 테스트

### 필수 검증

* 문서 저장 실패 시 작성 내용이 보존된다.
* 문서 버전이 정상 생성된다.
* 이전 버전 복원이 가능하다.
* 삭제 또는 보관 문서는 일반 목록에서 제외된다.
* 공유 범위 변경이 즉시 반영된다.
* AI 생성 문서가 일반 문서와 동일하게 관리된다.

---

## 6.4 AI 기능 테스트

### 필수 검증

* 문서 유형별 기본 구조가 유지된다.
* 템플릿이 생성 결과에 반영된다.
* 참고 문서가 생성 결과에 반영된다.
* 업무 기록과 To-do가 보고서 생성에 반영된다.
* AI 생성 실패 시 재시도가 가능하다.
* AI 결과는 자동 공유되지 않는다.
* AI 결과 저장 시 Draft 문서가 생성된다.
* AI 요청 로그가 기록된다.

---

## 6.5 검색 테스트

### 필수 검증

* 제목 검색이 동작한다.
* 본문 검색이 동작한다.
* 필터 검색이 동작한다.
* 원문 문서 이동이 동작한다.
* AI 요약 결과에 출처가 표시된다.
* 출처 없는 답변을 제한한다.
* 권한 변경 후 검색 결과와 원문 접근이 차단된다.

---

# 7. 품질 기준

## 7.1 사용성 기준

* 신규 사용자가 별도 교육 없이 첫 문서를 만들 수 있어야 한다.
* AI 문서 생성 흐름은 3-5분 안에 완료 가능해야 한다.
* 저장, 생성, 검색 상태가 사용자에게 명확히 보여야 한다.
* 오류 발생 시 사용자가 다음 행동을 알 수 있어야 한다.
* 관리자 화면에서 위험 작업은 확인 단계를 거쳐야 한다.

---

## 7.2 안정성 기준

* 로그인 상태가 예측 가능하게 유지되어야 한다.
* 문서 저장 중 오류가 발생해도 입력 내용이 사라지지 않아야 한다.
* AI 생성 실패가 전체 서비스 장애로 이어지지 않아야 한다.
* 검색 요약 실패 시 일반 검색 결과는 유지되어야 한다.
* 관리자 권한 변경은 즉시 반영되어야 한다.

---

## 7.3 보안 기준

* 조직별 데이터가 분리되어야 한다.
* 권한 없는 문서 접근은 모든 경로에서 차단되어야 한다.
* 검색과 AI 생성에도 동일한 권한 기준이 적용되어야 한다.
* 활동 로그에 민감한 본문이 과도하게 저장되지 않아야 한다.
* 초대 링크 만료와 취소가 정상 동작해야 한다.
* 마지막 Admin 제거가 차단되어야 한다.

# 9. 운영 정책 초안

## 9.1 사용자 운영 정책

* 조직 Admin이 사용자 초대와 비활성화를 관리한다.
* 비활성화된 사용자는 로그인할 수 없다.
* 사용자가 작성한 문서는 비활성화 후에도 조직 데이터로 유지된다.
* 마지막 Admin은 제거할 수 없다.

---

## 9.2 문서 운영 정책

* 문서는 조직 단위로 분리한다.
* 삭제된 문서는 일정 기간 복구 가능하게 유지한다.
* 외부 공유는 초기 출시에서 제공하지 않는다.
* 문서 공유 변경은 활동 로그에 기록한다.

---

## 9.3 AI 운영 정책

* AI 결과는 사용자가 검토한 뒤 저장한다.
* AI 결과는 자동으로 공유되지 않는다.
* AI 요청은 품질 개선과 문제 추적을 위해 기록한다.
* 민감한 본문 전체를 로그에 저장할지는 별도 정책으로 제한한다.
* 조직별 사용량을 관찰한다.

---

## 9.4 검색 운영 정책

* 검색 결과는 사용자 권한 범위 안에서만 제공한다.
* AI 요약 검색은 출처가 있는 경우에만 제공한다.
* 검색 로그는 보존 기간과 조회 권한을 제한한다.
* 검색 피드백은 품질 개선 용도로 사용한다.

* 샘플 템플릿 준비
* 오류 로그 확인 경로 준비
* 초기 사용자 초대 목록 준비

* 조직 데이터 혼선

# 14. 완료 기준

H단계는 다음 기준을 만족하면 완료로 본다.

* 핵심 사용자 흐름 테스트가 통과한다.
* 권한과 보안 점검이 통과한다.
* 문서 저장, AI 생성, 검색의 주요 오류가 해결되었다.
* 출시 전 점검표가 완료되었다.
* 알려진 이슈 목록이 정리되었다.
* 초기 사용자 피드백 수집 방식이 준비되었다.
* 운영 정책 초안이 준비되었다.
* 배포 후 확인 항목과 롤백 기준이 정리되었다.
