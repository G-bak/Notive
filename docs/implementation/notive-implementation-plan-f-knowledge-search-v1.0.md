# F. 사내 지식 검색 세부 구현 계획서 v1.0

# Notive 사내 지식 검색

---

# 1. 문서 목적

본 문서는 Notive 전체 구현 계획 중 F단계인 사내 지식 검색 구현의 세부 계획을 정의한다.

F단계의 목적은 사용자가 접근 권한을 가진 문서와 업무 맥락 데이터를 빠르게 찾고, 필요한 경우 AI 요약과 출처를 통해 내용을 이해할 수 있도록 하는 것이다.

이 단계는 Notive에 축적된 문서와 업무 기록을 조직의 지식 자산으로 활용하기 위한 검색 기반을 만든다.

---

# 2. F단계 목표

F단계가 완료되면 다음이 가능해야 한다.

* 사용자가 문서 제목과 본문을 검색할 수 있다.
* 문서 유형, 태그, 작성자, 팀, 날짜로 검색 결과를 필터링할 수 있다.
* 사용자가 자연어로 문서를 찾을 수 있다.
* 검색 결과는 사용자의 권한 범위 안에서만 표시된다.
* 검색 결과에서 원문 문서로 이동할 수 있다.
* AI 검색 결과에는 요약과 출처가 표시된다.
* 업무 다이어리와 To-do는 제한적으로 검색 또는 참고 결과에 포함할 수 있다.
* 검색 사용 기록과 품질 개선을 위한 기본 피드백을 수집할 수 있다.

# 5. 주요 사용자 흐름

## 5.1 기본 문서 검색

### 기본 흐름

1. 사용자가 검색 화면에 진입한다.
2. 검색어를 입력한다.
3. 시스템이 접근 가능한 문서에서 제목과 본문을 검색한다.
4. 검색 결과 목록을 표시한다.
5. 사용자가 결과를 선택한다.
6. 원문 문서 상세로 이동한다.

### 예외 흐름

* 검색어가 비어 있으면 최근 문서 또는 추천 필터를 표시한다.
* 결과가 없으면 검색어 수정 안내와 필터 초기화를 제공한다.
* 권한 없는 문서는 결과에 표시하지 않는다.
* 삭제된 문서는 기본 결과에서 제외한다.

---

## 5.2 필터 기반 검색

### 기본 흐름

1. 사용자가 검색어를 입력한다.
2. 문서 유형, 태그, 작성자, 팀, 날짜 필터를 선택한다.
3. 시스템이 필터 조건과 권한 조건을 함께 적용한다.
4. 결과를 관련도 또는 최근 수정순으로 정렬한다.
5. 사용자가 필요한 문서를 연다.

### 설계 기준

* 필터는 검색 결과를 좁히는 데 사용한다.
* 선택된 필터는 화면에서 명확히 보여야 한다.
* 필터 초기화가 쉬워야 한다.
* 권한 없는 필터 대상은 표시하지 않는다.

---

## 5.3 자연어 검색

### 기본 흐름

1. 사용자가 “작년 보안 정책 문서 찾아줘”처럼 자연어로 입력한다.
2. 시스템이 검색 의도를 파악한다.
3. 접근 가능한 문서에서 관련 결과를 찾는다.
4. 관련 문서 목록과 요약을 표시한다.
5. 사용자는 출처 문서를 열어 원문을 확인한다.

### 예외 흐름

* 질문이 모호하면 검색 결과 중심으로 보여주고 추가 입력을 유도한다.
* 충분한 근거 문서가 없으면 답변을 단정하지 않는다.
* 검색 가능한 권한 문서가 없으면 결과 없음으로 안내한다.

---

## 5.4 AI 요약형 검색

### 기본 흐름

1. 사용자가 자연어 질문을 입력한다.
2. 시스템이 관련 문서를 검색한다.
3. 관련 문서 일부를 근거로 요약 답변을 생성한다.
4. 답변 아래에 사용된 문서 출처를 표시한다.
5. 사용자가 출처 문서를 열어 확인한다.

### 설계 기준

* AI 답변은 원문 문서를 대체하지 않는다.
* 답변에는 출처가 반드시 표시되어야 한다.
* 출처 없는 답변은 제공하지 않거나 제한적으로 제공한다.
* 권한 없는 문서는 요약 근거로 사용하지 않는다.

---

## 5.5 검색 결과를 AI 문서 생성에 사용

### 기본 흐름

1. 사용자가 검색 결과에서 참고할 문서를 선택한다.
2. “AI 문서 생성에 사용”을 선택한다.
3. 선택 문서가 D단계 AI 생성의 참고 자료로 전달된다.
4. 사용자는 문서 유형과 요청 내용을 입력한다.
5. AI가 선택된 문서를 참고하여 초안을 생성한다.

### 설계 기준

* 선택 가능한 문서는 접근 권한이 있는 문서로 제한한다.
* 검색 결과에서 선택된 문서는 AIReference로 연결한다.
* AI 생성 시점에 권한을 다시 확인한다.

# 9. 권한 및 보안 기준

## 9.1 검색 권한 원칙

* 사용자는 접근 권한이 있는 대상만 검색할 수 있다.
* 검색 결과에서 권한 없는 문서의 존재를 암시하지 않는다.
* AI 요약은 권한 있는 출처만 사용한다.
* 출처 문서 접근 시에도 권한을 다시 확인한다.
* 검색 인덱스에는 조직 분리 기준을 반드시 포함한다.

---

## 9.2 대상별 권한 기준

| 대상 | 권한 기준 |
| --- | --- |
| Document | 문서 공유 범위와 사용자 역할 |
| DiaryEntry | 공개 범위와 작성자/팀 |
| Todo | 담당자, 생성자, 팀, 연결 문서 권한 |

---

## 9.3 검색 로그 보안

검색 로그는 운영과 품질 개선에 유용하지만 민감 정보를 포함할 수 있다.

### 기준

* 일반 사용자는 자신의 최근 검색어만 볼 수 있다.
* 관리자는 집계 수준의 검색 통계 위주로 확인한다.
* 민감한 검색어 원문 조회는 제한한다.
* 보존 기간을 설정한다.

---

# 10. AI 요약 검색 기준

## 10.1 AI 요약 생성 조건

AI 요약은 다음 조건을 만족할 때 제공한다.

* 관련 문서가 충분히 검색됨
* 사용자가 해당 문서에 접근 가능
* 출처를 표시할 수 있음
* 답변이 원문 근거를 기반으로 함

---

## 10.2 AI 요약 제한 기준

다음 경우에는 답변 생성을 제한한다.

* 관련 문서가 없음
* 출처를 특정할 수 없음
* 사용자가 접근 가능한 근거가 부족함
* 질문이 지나치게 모호함
* 권한상 민감한 문서가 섞여 있음

---

## 10.3 답변 UX 기준

* 답변은 요약 형태로 제공한다.
* 답변 아래에 출처 문서를 표시한다.
* 출처 문서로 바로 이동할 수 있어야 한다.
* 답변이 불확실하면 확인 필요로 표시한다.
* 사용자가 답변에 피드백을 남길 수 있다.

---

# 11. AI 문서 생성과의 연결

검색 결과는 D단계 AI 문서 생성의 참고 자료로 사용할 수 있다.

## 연결 흐름

1. 사용자가 검색한다.
2. 검색 결과에서 문서를 선택한다.
3. AI 문서 생성에 사용을 선택한다.
4. 선택 문서가 AIReference에 추가된다.
5. AI 생성 화면으로 이동한다.
6. 생성 요청 시 권한을 다시 확인한다.

---

# 12. 오류 및 예외 처리

## 12.1 검색 오류

* 검색 서비스 오류
* 검색어 처리 실패
* 필터 조건 오류
* 인덱스 지연

### UX 기준

* 검색어와 필터 입력은 유지한다.
* 재시도 버튼을 제공한다.
* 검색 결과 없음과 시스템 오류를 구분한다.

---

## 12.2 권한 변경 오류

* 검색 결과 표시 후 문서 권한이 변경됨
* 출처 문서가 삭제됨
* 문서가 보관 처리됨

### 대응 기준

* 원문 열기 시 권한을 다시 확인한다.
* 접근 불가 상태를 안내한다.
* AI 요약 출처가 더 이상 접근 불가하면 표시를 제한한다.

---

## 12.3 AI 요약 오류

* 요약 생성 실패
* 관련 출처 부족
* 답변 생성 시간 초과
* 출처 문서 처리 실패

### 대응 기준

* 문서 검색 결과는 유지한다.
* 요약만 실패했음을 명확히 표시한다.
* 사용자가 원문 결과를 계속 탐색할 수 있게 한다.

---

# 13. 테스트 계획

## 13.1 핵심 테스트 시나리오

| 시나리오 | 검증 내용 |
| --- | --- |
| 제목 검색 | 제목 기반 문서 결과 표시 |
| 본문 검색 | 본문 포함 문서 결과 표시 |
| 필터 검색 | 유형, 태그, 작성자, 날짜 필터 적용 |
| 결과 정렬 | 관련도순/최근 수정순 정렬 |
| 원문 이동 | 결과에서 문서 상세 이동 |
| 자연어 검색 | 질문형 검색 결과 표시 |
| AI 요약 | 요약과 출처 표시 |
| 결과 없음 | 빈 상태 안내 |
| 검색 로그 | 검색어와 결과 수 기록 |
| 검색 피드백 | 만족/불만족 저장 |

---

## 13.2 권한 테스트

* Private 문서가 작성자에게만 검색되는지 확인
* Team 문서가 같은 팀 사용자에게만 검색되는지 확인
* 다른 조직 문서가 검색되지 않는지 확인
* 권한 없는 문서가 AI 요약 출처로 사용되지 않는지 확인
* 원문 이동 시 권한을 다시 확인하는지 확인
* Private 업무 기록이 작성자 검색에만 노출되는지 확인

---

## 13.3 AI 요약 테스트

* 출처가 있는 경우에만 요약이 제공되는지 확인
* 출처 문서 목록이 표시되는지 확인
* 요약 실패 시 일반 검색 결과가 유지되는지 확인
* 모호한 질문에서 단정적인 답변을 피하는지 확인
* 권한 변경 후 출처 접근이 차단되는지 확인

---

# 14. 완료 기준

F단계는 다음 기준을 만족하면 완료로 본다.

* 사용자가 통합 검색 화면에서 문서를 검색할 수 있다.
* 제목과 본문 기반 검색이 동작한다.
* 문서 유형, 태그, 작성자, 팀, 날짜 필터를 사용할 수 있다.
* 검색 결과에서 원문 문서로 이동할 수 있다.
* 권한 없는 문서는 검색 결과에 노출되지 않는다.
* 자연어 검색으로 관련 문서를 찾을 수 있다.
* AI 요약 결과에는 출처가 표시된다.
* 출처 문서는 사용자 권한 범위 안에서만 사용된다.
* 검색 결과를 AI 문서 생성 참고 자료로 넘길 수 있다.
* 검색 로그와 기본 피드백이 기록된다.

# 15. 주요 리스크 및 대응

| 리스크 | 설명 | 대응 방향 |
| --- | --- | --- |
| 권한 누락 | 검색 결과나 AI 요약에 민감 문서가 노출될 수 있음 | 검색 전후 권한 검사와 조직 분리 기준 강화 |
| 검색 품질 부족 | 사용자가 원하는 문서를 찾지 못할 수 있음 | 필터, 정렬, 피드백으로 점진 개선 |
| AI 요약 신뢰도 | 출처 없는 답변이나 과도한 추론 가능 | 출처 기반 요약만 제공 |
| 인덱스 지연 | 최신 문서가 검색되지 않을 수 있음 | 수정 시 인덱스 갱신 정책 정의 |
| 검색 로그 민감성 | 검색어 자체가 민감 정보일 수 있음 | 로그 보존 기간과 조회 권한 제한 |
| 범위 확대 | 검색이 지식 그래프/추천 시스템으로 비대화 가능 | MVP는 문서 검색과 출처 표시 중심으로 제한 |

F단계 완료 후 G단계 관리자 및 운영 기능 구현으로 넘어간다.
