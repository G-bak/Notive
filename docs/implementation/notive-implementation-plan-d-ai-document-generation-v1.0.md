# D. AI 문서 생성 세부 구현 계획서 v1.0

# Notive AI 문서 생성

---

# 1. 문서 목적

본 문서는 Notive 전체 구현 계획 중 D단계인 AI 문서 생성 구현의 세부 계획을 정의한다.

D단계의 목적은 사용자가 자연어 요청, 문서 유형, 템플릿, 참고 자료를 기반으로 업무 문서 초안을 생성하고, 생성 결과를 검토한 뒤 문서로 저장할 수 있는 흐름을 구현하는 것이다.

이 단계는 Notive의 핵심 가치인 “업무 맥락을 반영한 문서 생성”의 첫 구현 단계다.

---

# 2. D단계 목표

D단계가 완료되면 다음이 가능해야 한다.

* 사용자가 문서 유형을 선택해 AI 문서 생성을 시작할 수 있다.
* 사용자가 템플릿을 선택하거나 기본 템플릿을 사용할 수 있다.
* 사용자가 자연어 요청을 입력할 수 있다.
* 사용자가 참고할 기존 문서를 선택할 수 있다.
* AI가 업무 문서 초안을 생성한다.
* 생성 결과를 미리보기에서 확인할 수 있다.
* 생성 결과를 문서 편집기로 넘겨 수정할 수 있다.
* 생성 결과를 Draft 또는 Active 문서로 저장할 수 있다.
* AI 생성 실패, 지연, 재시도 흐름이 제공된다.
* AI 요청과 결과 상태가 추적 가능하다.

# 5. 주요 사용자 흐름

## 5.1 기본 AI 문서 생성

### 기본 흐름

1. 사용자가 AI 문서 생성 메뉴에 진입한다.
2. 문서 유형을 선택한다.
3. 템플릿을 선택한다.
4. 자연어 요청을 입력한다.
5. 생성 버튼을 누른다.
6. AI 생성 진행 상태를 확인한다.
7. 생성 결과 미리보기를 확인한다.
8. 편집기로 이동한다.
9. 필요한 내용을 수정한다.
10. 문서로 저장한다.

### 예외 흐름

* 요청 내용이 비어 있으면 입력 안내를 표시한다.
* 생성 실패 시 재시도와 요청 수정 옵션을 제공한다.
* 생성 시간이 길어지면 진행 상태를 명확히 표시한다.
* 생성 결과가 불만족스러우면 재생성 또는 직접 편집을 선택한다.

---

## 5.2 템플릿 기반 문서 생성

### 기본 흐름

1. 사용자가 문서 유형을 선택한다.
2. 시스템이 사용 가능한 템플릿을 보여준다.
3. 사용자가 템플릿을 선택한다.
4. 템플릿의 구조와 작성 기준이 AI 요청에 반영된다.
5. AI가 템플릿 형식에 맞춘 초안을 생성한다.
6. 사용자는 결과를 편집기로 넘겨 저장한다.

### 예외 흐름

* 사용 가능한 템플릿이 없으면 기본 구조를 사용한다.
* 템플릿 접근 권한이 없으면 목록에 표시하지 않는다.
* 템플릿이 비활성화된 경우 선택할 수 없다.

---

## 5.3 기존 문서 참고 생성

### 기본 흐름

1. 사용자가 참고 자료 선택 영역을 연다.
2. 접근 가능한 문서 목록에서 참고 문서를 선택한다.
3. 선택된 문서가 생성 요청에 포함된다.
4. AI는 참고 문서를 반영해 초안을 생성한다.
5. 결과 미리보기에서 참고된 자료를 확인한다.
6. 사용자가 결과를 편집하고 저장한다.

### 예외 흐름

* 권한 없는 문서는 선택 목록에 표시하지 않는다.
* 참고 문서가 너무 많으면 선택 개수를 제한한다.
* 삭제 또는 보관된 문서는 기본 선택 대상에서 제외한다.
* 참고 문서가 생성 중 삭제되면 해당 문서를 제외하고 진행한다.

---

## 5.4 AI 생성 결과 재시도

### 기본 흐름

1. 사용자가 생성 결과를 확인한다.
2. 결과가 만족스럽지 않으면 재생성을 선택한다.
3. 기존 요청을 유지하거나 수정한다.
4. AI가 다시 생성한다.
5. 사용자는 새 결과를 선택해 편집기로 이동한다.

### 설계 기준

* 재생성은 기존 문서를 자동으로 덮어쓰지 않는다.
* 재생성 결과는 사용자가 선택하기 전까지 초안 후보로 유지한다.
* 재생성 횟수 제한 여부는 운영 정책에 따라 결정한다.

---

## 5.5 AI 생성 결과 저장

### 기본 흐름

1. 사용자가 생성 결과에서 “문서로 저장” 또는 “편집하기”를 선택한다.
2. 시스템이 Draft 문서를 생성한다.
3. 문서 편집 화면으로 이동한다.
4. 사용자가 수정 후 저장한다.
5. 저장된 문서는 일반 문서와 동일하게 관리된다.

### 저장 기준

* AI 생성 결과는 처음에는 초안으로 취급한다.
* 저장 후 일반 문서와 동일한 권한, 버전, 공유 정책을 따른다.
* AI 생성 방식과 요청 정보를 문서 메타데이터에 연결할 수 있다.

# 8. AI 생성 요청 구성

## 8.1 요청 구성 요소

AI 생성 요청은 다음 요소를 조합한다.

* 사용자 요청
* 문서 유형
* 선택 템플릿
* 참고 문서 요약 또는 본문 일부
* 조직 기본 문체
* 출력 형식 기준
* 금지 또는 주의 사항

---

## 8.2 문서 유형별 기본 구조

| 문서 유형 | 기본 구조 예시 |
| --- | --- |
| 업무 보고서 | 요약, 진행 업무, 이슈, 다음 계획 |
| 회의록 | 회의 개요, 참석자, 논의 내용, 결정 사항, 액션 아이템 |
| 제안서 | 배경, 문제 정의, 제안 내용, 기대 효과, 일정 |
| 기획서 | 목적, 대상, 주요 기능, 일정, 리스크 |
| SOP | 목적, 적용 범위, 절차, 주의사항, 책임자 |
| 이메일 초안 | 수신자 맥락, 핵심 메시지, 요청 사항, 마무리 |

---

## 8.3 생성 품질 기준

AI 생성 결과는 다음 기준을 만족해야 한다.

* 선택한 문서 유형에 맞는 구조를 가진다.
* 요청 내용과 무관한 내용을 과도하게 만들지 않는다.
* 참고 자료가 있을 경우 그 맥락을 반영한다.
* 회사 업무 문서에 맞는 간결한 문체를 사용한다.
* 사용자가 바로 수정할 수 있는 형태로 제공된다.
* 불확실한 내용은 단정하지 않도록 한다.

# 9. 권한 및 보안 기준

## 9.1 AI 생성 권한

| 역할 | AI 문서 생성 |
| --- | --- |
| Viewer | 불가 |
| Editor | 가능 |
| Manager | 가능 |
| Admin | 가능 |

---

## 9.2 참고 자료 권한

AI 생성에 사용할 수 있는 참고 자료는 사용자가 접근 가능한 자료로 제한한다.

### 원칙

* 권한 없는 문서는 선택 목록에 표시하지 않는다.
* 직접 ID를 조작해도 참고 자료로 사용할 수 없다.
* 생성 시점의 참고 자료 권한을 기록한다.
* 생성 이후 권한이 변경되더라도 생성 기록은 감사 목적의 메타데이터만 유지한다.

---

## 9.3 결과 접근 권한

* AI 생성 결과는 요청 사용자만 기본 접근 가능하다.
* 문서로 저장된 후에는 문서 공유 정책을 따른다.
* 저장 전 결과를 자동으로 팀에 공유하지 않는다.
* Admin이 AI 생성 기록을 어디까지 볼 수 있는지는 운영 정책에 따라 결정한다.

# 10. 저장 및 문서 연결

## 10.1 저장 방식

AI 결과 저장은 C단계 문서 저장 흐름을 사용한다.

### 저장 옵션

* Draft로 저장
* 편집기로 이동 후 저장
* 기존 문서에 덮어쓰기 또는 삽입은 MVP 이후 검토

---

## 10.2 문서 메타데이터 연결

AI로 생성된 문서는 다음 정보를 가질 수 있다.

* 생성 방식: AI
* AI 요청 ID
* 사용 템플릿 ID
* 문서 유형
* 참고 자료 목록
* 최초 생성 사용자

---

## 10.3 버전 기록

* AI 결과를 문서로 저장하면 최초 버전을 생성한다.
* 사용자가 편집 후 저장하면 새로운 버전을 생성한다.
* 재생성 결과는 기존 문서를 자동으로 변경하지 않는다.

# 11. 오류 및 예외 처리

## 11.1 생성 실패

* 요청 수정 후 다시 생성할 수 있게 한다.
* 사용자의 입력값은 유지한다.

* 취소 후 다시 요청할 수 있게 한다.
* 중복 요청을 방지한다.

# 13. 테스트 계획

## 13.1 핵심 테스트 시나리오

| 시나리오 | 검증 내용 |
| --- | --- |
| 기본 생성 | 요청 입력 후 문서 초안 생성 |
| 템플릿 생성 | 선택 템플릿 구조 반영 |
| 참고 문서 생성 | 선택 문서 맥락 반영 |
| 결과 미리보기 | 생성 결과와 메타데이터 표시 |
| 편집 전환 | 생성 결과가 편집기에 반영 |
| Draft 저장 | 생성 결과를 초안 문서로 저장 |
| 재생성 | 기존 요청 기반 재시도 |
| 생성 실패 | 실패 안내와 재시도 제공 |
| 권한 제한 | 권한 없는 참고 문서 사용 차단 |
| 로그 기록 | AI 요청과 결과 상태 기록 |

---

## 13.2 권한 테스트

* Viewer가 AI 생성 화면에 접근할 수 없는지 확인
* Editor가 접근 가능한 문서만 참고 자료로 선택할 수 있는지 확인
* 다른 조직 문서를 참고 자료로 사용할 수 없는지 확인
* 생성 결과가 요청 사용자에게만 보이는지 확인
* 저장 후 문서 권한 정책이 적용되는지 확인

---

## 13.3 품질 테스트

* 문서 유형별 기본 구조가 유지되는지 확인
* 템플릿 선택 시 출력 구조가 반영되는지 확인
* 요청 내용과 무관한 결과가 과도하게 생성되지 않는지 확인
* 참고 문서가 있을 때 출처 정보가 표시되는지 확인
* 생성 결과가 편집 가능한 문서 형태인지 확인

---

# 14. 완료 기준

D단계는 다음 기준을 만족하면 완료로 본다.

* Editor 이상 사용자가 AI 문서 생성을 요청할 수 있다.
* 문서 유형과 템플릿을 선택할 수 있다.
* 자연어 요청을 입력해 문서 초안을 생성할 수 있다.
* 접근 가능한 기존 문서를 참고 자료로 선택할 수 있다.
* 생성 중, 성공, 실패 상태가 사용자에게 표시된다.
* 생성 결과를 미리보기에서 확인할 수 있다.
* 생성 결과를 편집기로 넘겨 수정할 수 있다.
* 생성 결과를 Draft 문서로 저장할 수 있다.
* AI 요청과 결과 상태가 기록된다.
* 권한 없는 문서는 AI 생성 맥락에 포함되지 않는다.

| 리스크 | 설명 | 대응 방향 |
| --- | --- | --- |

| 생성 지연 | 응답 시간이 길어 사용자 이탈 가능 | 진행 상태, 취소, 재시도 제공 |
| 권한 우회 | 권한 없는 문서가 참고 자료로 들어갈 수 있음 | 선택 단계와 요청 처리 단계에서 모두 권한 검사 |
| 비용 증가 | 재생성과 반복 요청으로 비용 증가 가능 | 사용량 기록과 제한 정책 준비 |
| 로그 보안 | 요청 본문에 민감 정보가 포함될 수 있음 | 로그 저장 범위 최소화 검토 |
| 결과 덮어쓰기 | 재생성 결과가 기존 문서를 훼손할 수 있음 | 사용자가 저장하기 전까지 문서 변경 금지 |

# 16. E단계 연결

D단계 완료 후 E단계 업무 맥락 기능 구현으로 넘어간다.

E단계 착수 전 준비되어야 할 항목은 다음과 같다.

* AI 생성 요청에 참고 자료를 연결하는 구조
* 참고 자료 유형 확장 가능성
* Draft 문서 저장 흐름
* AI 결과 미리보기와 편집 전환 흐름
* AI 요청 로그
* 문서 유형별 기본 생성 구조
* 권한 기반 참고 자료 제한
